What is Hexo?

Hexo is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.
… 원문보기

Hexo는 빠르고 간단하며 파워풀한 블로그 프레임워크입니다. Markdown(혹은 다른 언어로) 포스트를 작성하면 Hexo는 불과 몇초 안에 예쁜 테마를 적용한 static 파일을 생성합니다.

1. 설치

Hexo를 설치하기 위해선 아래 두 소프트웨어가 먼저 설치되어 있어야 합니다.

  • Node.js
  • Git

Node.js는 크롬의 V8 자바스크립트 엔진으로 빌드된 자바스크립트 실행환경입니다. 확장가능한 네트워크 어플리케이션을 위해 설계되었지만 현재에 이르러서는 그 이상의 기능을 수행할 수 있습니다.

Git은 작은 파일에서부터 큰 프로젝트에 이르는 모든것들의 버전을 관리하기 위한 프로그램입니다.

두 소프트웨어가 모두 준비되었다면 아래 명령어를 터미널에 입력하여 Hexo를 설치할 수 있습니다.

만약 우분투나 맥을 사용하고 있다면 권한 상승을 위해 sudo 명령어를 사용해야합니다.

2. HEXO 폴더 초기화

Hexo를 설치했다면, 아래 명령어를 실행하여 실제로 Hexo를 사용하기위한 폴더를 생성하세요. <folder>는 사용하고 싶은 폴더이름으로 바꾸어 입력하시면 됩니다.

이제 만들어진 폴더로 이동한 뒤, 아래 명령어를 실행하여 Hexo 실행을 위해 필요한 npm 패키지를 설치합니다.

설치가 끝났다면 프로젝트 폴더는 아래와 같은 형태를 가지고 있습니다.

각각의 파일(혹은 폴더)는 아래의 역할을 합니다.

1. _config.yml
Hexo 설정을 위한 파일입니다. 다양한 설정을 할 수 있습니다.

2. _package.json
Node프로젝트에 관한 정보, npm 패키지에 관한정보가 담겨있습니다. 기본적으로 EJS, Stylus, Makrdown렌더링을 위한 패키지가 설치됩니다.

3. scaffolds
새 포스트를 작성할 때 사용하기 위한 기본 틀을 가진 파일을 모아둔 폴더입니다.

4. source
사이트의 내용이 위치하는 폴더입니다. Hexo는 hidden된 파일과 폴더 혹은 _(underscore) 로 시작하는 파일과 폴더는 무시합니다(당연히 _post 폴더는 무시하지 않습니다.). 렌더링이 가능한 파일(Markdown 파일 등)은 처리가 된 뒤 public 폴더에 들어가고, 다른 파일은 단순히 복사됩니다.

5. themes
테마 파일이 위치한 폴더입니다. Hexo는 사이트의 컨텐츠에 테마를 입혀 정적 웹 사이트를 생성합니다.
